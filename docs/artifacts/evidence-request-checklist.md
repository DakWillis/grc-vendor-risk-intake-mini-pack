# Evidence Request Checklist

## Low risk (minimum)
- Security overview (1â€“2 pages)
- Data handling statement
- Incident notification policy or summary

## Medium risk (standard)
- SOC 2 Type II (preferred) or equivalent report/attestation
- Vulnerability management summary
- Access control and MFA statement
- Encryption at rest/in transit confirmation
- Subprocessor list (if applicable)

## High risk (enhanced)
- SOC 2 Type II or ISO 27001 certificate + Statement of Applicability (if available)
- Pen test executive summary (recent)
- BCP/DR summary and RTO/RPO targets
- Secure SDLC statement (if software vendor)
- Incident response plan summary
- Data deletion/return procedure
